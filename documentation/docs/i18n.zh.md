# 国际化（i18n）

Pinmok 完全延续 Django 的多语言体系，不引入额外的配置层。`padmin` 已内置中文（`zh-hans`）和英文（`en`）翻译，开发者按 Django 标准流程为自己的 app
添加翻译即可。

除标准 i18n 支持外，Pinmok 还提供了两个工具：

- **多语言模型**（`TranslatableModel` / `TranslationModel`）：用于主表+翻译表的数据库级多语言场景
- **PO 去重工具**（`dedupe_po`）：清理 `.po` 文件中的重复条目

---

## 配置

在 `settings.py` 中启用标准 Django i18n 配置：

```python
USE_I18N = True
LANGUAGE_CODE = 'zh-hans'  # 默认语言

LANGUAGES = [
    ('zh-hans', '简体中文'),
    ('en', 'English'),
]
```

在 `MIDDLEWARE` 中加入 `LocaleMiddleware`，位置必须在 `SessionMiddleware` 之后、`CommonMiddleware` 之前：

```python
MIDDLEWARE = [
    ...
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.locale.LocaleMiddleware',  # ← 这里
    'django.middleware.common.CommonMiddleware',
    ...
]
```

如需通过 URL 前缀切换语言（如 `/zh-hans/`、`/en/`），在 `urls.py` 中使用 `i18n_patterns`：

```python
from django.conf.urls.i18n import i18n_patterns

urlpatterns += i18n_patterns(
    path('', include('your_app.urls')),
)
```

`padmin` 的后台界面语言跟随 Django 当前语言环境，无需额外配置。

## 多语言模型

适用场景：内容数据需要多语言版本，例如分类名称、文章标题、导航标签等。采用**主表 + 翻译表**的结构，主表存储与语言无关的字段，翻译表存储各语言的文字内容。

### 定义模型

主表继承 `TranslatableModel`，翻译表继承 `TranslationModel`：

```python
from pinmok.core.models import TranslatableModel, TranslationModel
from pinmok.core.constants import TRANSLATION_RELATED_NAME
from django.db import models


class Category(TranslatableModel):
    sort_order = models.IntegerField(default=0)
    is_active = models.BooleanField(default=True)

    class Meta:
        ordering = ['sort_order']


class CategoryTranslation(TranslationModel):
    category = models.ForeignKey(
        Category,
        on_delete=models.CASCADE,
        related_name=TRANSLATION_RELATED_NAME,  # 必须为 'translations'
    )
    name = models.CharField(max_length=255)
    description = models.CharField(max_length=200, blank=True, default='')

    class Meta(TranslationModel.Meta):
        constraints = [
            models.UniqueConstraint(
                fields=['category', 'language'],
                name='uniq_category_language'
            )
        ]

    def get_display_text(self):  # 必须实现
        return self.name
```

**注意事项：**

- 翻译表的 FK `related_name` 必须为 `'translations'`，建议使用常量 `TRANSLATION_RELATED_NAME` 而非硬编码字符串
- `UniqueConstraint` 必须定义，防止同一语言重复写入
- `get_display_text()` 必须实现，框架的 `__str__` 和 `translation` 属性依赖它

### 框架自动处理的部分

继承后，以下功能开箱即用，无需额外编写：

**获取当前语言翻译：**

```python
category = Category.objects.get(pk=1)
category.translation.name  # 自动返回当前语言，找不到时按优先级 fallback
```

fallback 优先级依次为：当前语言 → 语言基础码（`zh-hans` → `zh`）→ `LANGUAGE_CODE` → 第一条可用翻译。

**指定语言：**

```python
category.get_translation('en').name
```

**`__str__` 自动代理：**

```python
str(category)  # 返回当前语言的 get_display_text()
```

### 开发者需要自己做的部分

- 定义翻译表的具体字段（`name`、`description` 等）
- 实现 `get_display_text()`
- 在 `admin.py` 中注册翻译表（通常用 `StackedInline` 或 `TabularInline`）：

```python
from django.contrib import admin
from .models import Category, CategoryTranslation


class CategoryTranslationInline(admin.StackedInline):
    model = CategoryTranslation
    extra = 1


@admin.register(Category)
class CategoryAdmin(admin.ModelAdmin):
    inlines = [CategoryTranslationInline]
```

### 解决 N+1 问题

在列表页或批量处理场景中，使用 `with_translations()` 预加载翻译数据：

```python
# 视图或序列化器中
categories = Category.with_translations()

# 也可以在现有 queryset 上使用
categories = Category.with_translations(
    Category.objects.filter(is_active=True)
)
```

预加载后，`translation` 属性和 `get_translation()` 不再触发额外查询。

### 更新翻译后刷新缓存

保存翻译记录后，如需在同一请求中立即读取新值，手动清除实例缓存：

```python
translation.save()
category.invalidate_translation_cache()
```

## PO 去重工具

Django 的 `makemessages` 在某些情况下会在 `.po` 文件中产生重复条目，`dedupe_po` 工具用于清理这些重复项，同时可以去除与 Django 内置翻译重复的条目（
`django.contrib.admin`、`django.contrib.auth` 等已有的翻译无需重复维护）。

### 用法

在项目根目录运行：

```bash
# 处理默认语言（zh-hans），所有 domain
python manage.py dedupe_po

# 指定语言
python manage.py dedupe_po -l en

# 只处理 django domain
python manage.py dedupe_po -d django

# 指定扫描目录
python manage.py dedupe_po --dir /path/to/project
```

工具会递归扫描项目目录下所有 app 的 `locale/<lang>/LC_MESSAGES/` 目录，统一处理，无需逐个 app 单独运行。

### 选项

| 选项              | 默认值       | 说明                                 |
|-----------------|-----------|------------------------------------|
| `-l, --lang`    | `zh-hans` | 目标语言代码                             |
| `-d, --domain`  | 全部        | 限定 domain，可选 `django` / `djangojs` |
| `--dir`         | 当前目录      | 指定扫描根目录                            |
| `--no-backup`   | —         | 不创建备份（默认会创建 `.bak` 文件）             |
| `--no-comments` | —         | 不保留注释                              |

### 备份

默认情况下，每次处理前会在同目录生成时间戳备份文件：

```
django_250601120000.bak
```

确认结果无误后可手动删除。如不需要备份，加 `--no-backup` 参数。
