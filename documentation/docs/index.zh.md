# Pinmok 1.0 开发文档

## 前言

Pinmok 是一款基于 Django Admin 的模块化后台管理框架，完全保留了 Django 的原有开发习惯，以模块化的设计理念对 Admin 进行了扩展。

Pinmok 允许开发者根据自身的需求，以应用的方式进行扩展。应用之间即可以保持独立，也可以通过系统协同工作，每个应用就是一个功能性模块。

Pinmok 的目标，是让开发者更自由，更快捷，让开发者只需要关注自己的业务实现，不需要再被 Admin 的局限所束缚。

### 设计理念

Django Admin 本身专注于模型的增删改查，而缺乏对复杂应用的处理。Pinmok
基于此进行了扩展，让后台管理模块化，开发者只需要安装或开发一个应用，就能增加相应的功能。最大程度减少代码的冗余，减少开发者的负担，降低学习成本，增加效率。

让功能模块自由组合，灵活扩展，一个系统只需要把模块都拼在一起即可，Pinmok 即拼模块。

### 适用人群

希望在 Django Admin 基础上快速搭建功能，需要快速构建应用管理功能的开发者。

### Pinmok 生态

Pinmok 还会提供一些免费的应用，例如：Pinmok Content，实现了轻量级的内容管理，支持模板建站，同时提供完备 API，适配前后端分离架构，可对接各类轻量应用与平台账号生态。

### 参与贡献

Pinmok 于 2025 年 4 月立项，由作者利用业余时间潜心打磨至今，耗时一年多反复推敲架构、优化逻辑，力求打造稳健规范的框架级精品，拒绝冗余杂乱的低效代码。

项目现已正式发布 1.0 版本，整体开发并未止步，后续会长期持续迭代优化。
