# Pinmok 1.0 Documentation

## Preface

Pinmok is a modular admin framework built upon Django Admin. It extends Django Admin following a modular design principle while fully
adhering to native Django development conventions.

Developers can expand system capabilities via independent apps according to actual demands. These apps can run standalone or interact with
one another seamlessly, with each serving as a fully self-contained functional module.

Pinmok aims to grant developers greater flexibility and development efficiency, allowing you to concentrate purely on business logic without
being restricted by the inherent limitations of vanilla Django Admin.

### Design Philosophy

Native Django Admin mainly targets model-level CRUD operations and falls short in supporting complex business scenarios. Pinmok optimizes
this paradigm by modularizing backend management capabilities. Developers can quickly add desired functions by installing or developing
dedicated apps. This approach effectively cuts redundant code, eases development burden, lowers learning barriers and boosts overall
development efficiency.

Functional modules can be freely assembled and extended flexibly — a complete business system can be easily composed by combining different
modules. Pinmok — pin the modules.

### Who Is It For

Developers who need to rapidly build backend management systems based on Django Admin, without repetitive foundational development work.

### Pinmok Ecosystem

Pinmok will maintain a rich ecosystem of free official extension apps.
For instance, Pinmok Content provides lightweight content management capabilities, supporting template-based traditional website
construction, full-standard APIs for front-end and back-end separation scenarios, as well as access adaptation for lightweight applications
and official account service ecosystems worldwide.

### Contributing

Pinmok was initiated in April 2025. It has been continuously polished and architecturally refined in the author's spare time for more than a
year. The project always sticks to building well-structured, high-quality framework-level products, and strictly avoids messy and
low-quality code implementation.

The official 1.0 version is now released, yet the development journey does not end here.

Currently, the project is still in the early stage of development, with plenty of functions and roadmap plans waiting to be implemented. If
you recognize the value and prospect of this project, you are warmly welcomed to join in and contribute together. For an individually-driven
open-source project, every Star you give in the repository is sincere encouragement and powerful motivation.
