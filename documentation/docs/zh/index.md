# 前言

Django Admin 是一个成熟的后台管理工具，模型注册进去，增删改查界面自动就有了。但它有一个根本性的局限：它是为模型设计的，边界止于模型。

如果你想在后台里加一个不依赖模型的功能页面，或者希望把不同功能模块拆开独立维护、按需组合，Admin 本身给不了这些。并非它做得不够好，而是它从未被设计用于这个目的。

Pinmok 做的事情很简单：突破这个边界。

它在 Django Admin 之上加了一层，让后台不再局限于模型管理。任何功能都可以以应用模块的形式接入——独立开发，独立安装，菜单和权限自动注册到系统里。需要什么就装什么，
不需要的拿掉就行，模块之间互不干扰。整个系统就是这样一块一块拼起来的，这也是 Pinmok 这个名字的由来：拼模块。

对开发者来说，Pinmok 的使用方式和 Django Admin 几乎一样，不需要重新学习一套新的体系，一两个小时就能上手。它完全遵循 Django 的设计习惯，不绑定版本，随时可以跟着
Django 升级，你的代码始终在自己手里。

Pinmok 是开源项目，MIT 许可，免费使用。官方模块将会持续更新，目前已有 Pinmok Content——一个极简的内容管理模块，支持模板建站，同时提供完整
API，适配前后端分离架构。你也可以自己开发模块，把它贡献给社区，或者只在自己的项目里使用。

适用场景： 企业内部管理系统、数据统计与监控后台、IoT 设备管理、爬虫任务调度，以及任何需要可扩展后台管理能力的 Django 项目。

本文档面向具备 Django 基础的开发者。从安装配置到开发第一个 Pinmok 应用，文档会一步步带你走完整个流程。每章开头提供速查表，熟悉之后可以直接按需查阅。
